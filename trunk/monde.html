Commandes de base

  
 En plus de comprendre le Javascript, Reeborg et tous les autres robots
  connaissent les instructions suivantes:



      
Veuillez prendre note: en Javascript, true et false sont écrits
      en lettre minuscules; en Python, True et False débutent avec une
      lettre majuscule.  Pour éviter de spécifier ceci à chaque fois ci-dessous, j'utiliserai
      "vrai" et "false" pour signifier la même chose.

      
Attention: Vous pouvez écrire des noms de commandes
      avec des accents en Javascript mais pas en Python.


      
Commandes de base

  
 En plus de comprendre le Javascript ou le Python, Reeborg et tous les autres robots
  connaissent les instructions suivantes:


  
    
    
  • au_but(): la condition est "vrai" si Reeborg a atteint son but.
    • 
    
  • au_but_orientation(): la condition est "vrai" si Reeborg a la bonne orientation.
    • 
    
  • construit_un_mur(): ajoute un mur tout juste devant la position de Reeborg
    • 
    
  • termine: indique qu'un programme doit se terminer avec cette instruction. 
    • 
    
  • rien_devant(): "vrai" s'il n'y a pas de mur bloquant la voie de Reeborg.
    • 
    
  • a_des_jetons(): "vrai" si Reeborg a au moins un jeton en main.
    • 
    
  • face_au_nord(): "vrai" si Reeborg fait face au nord (vers le haut de l'écran).
    • 
    
  • avance()
    • 
    
  • depose() ou depose("objet") où "objet" est l'un de "jeton",
      "triangle", "carré" ou "étoile": Reeborg dépose l'objet
      en question là où il se trouve.
    • 
      
  • objet_ici(): "false" s'il n'y a aucun objet à cet endroit, autrement on retourne
      le nom de l'objet.
    • 
    
  • rien_a_droite(): "vrai" s'il n'y a pas de mur immédiatement à la droite de Reeborg.
    • 
    
  • repete(f, n): excute f() n fois.
    • 
    
  • prend() ou prend("objet") où "objet" est l'un de "jeton",
      "triangle", "carré" ou "étoile"
    • 
    
  • jeton_ici(): "vrai" s'il y a un ou des jetons à cet endroit.
    • 
    
  • tourne_a_gauche()
    • 
    
  • dis(message): "message" apparait dans un dialogue.
    • 
  



  
Pour créer des robots, on peut utiliser RobotUsage() prédédé de new
  si on utilise Javascript.

  
De plus, les commandes suivantes sont généralement disponibles sans être liées
    à un robot en particulier. 

  
    
        
  • import_biblio(): pour Javascript, inclut le code qui se trouve sous l'onglet bibliothèque
    • 
        
  • from biblio import *: pour Python, iinclut le code qui se trouve sous l'onglet bibliothèque
    • 
    
  • examine(objet Javascript)
    • 
    
  • pause(); peut être invoqué avec un argument en millisecondes indiquant la durée de
    la pause dans quel cas le programme continue sans avoir à cliquer sur un bouton. 
    • 
        
  • son("vrai"): ajoute des effets sonores. Ces effets sonores peuvent être arrêtés en utilant "false" comme
                  argument ou omettant l'argument comme ceci: son().
                  Les effets sonores sont arrêtés par défaut ou si le délai entre les actions est inférieur à 250ms.
                  
    • 
    
  • pense(nombre): spécifie le montant de temps (en millisecondes) entre chaque action d'un robot. 
    • 
    
  • ecrit(chaîne): écrit une chaîne de caractère dans le journal de Reeborg, par exemple ecrit("Bonjour !"),
      mais seulement si le monde est visible - autrement utiliser print. 
    • 
    
  • selectionne_monde(chaîne): sélectionne un monde donné. 
    • 
    
  • selectionne_defi(chaîne): sélectionne un monde (caché) donné. 
    • 
        
  • print(chaîne): seulement valide avec Python; semblable à write.
    • 

  


 
Pour en savoir davantage, consultez les leçons.



     
À propos de Reeborg


    
Inspiré par Guido van Robot, lui-même inspiré par le robot Karel conçu
  par Richard Pattis, Reeborg est apparu en premier dans RUR-PLE, un logiciel
  que j'ai créé pour faciliter l'apprentissage de Python.



Reeborg est votre guide dans l'apprentissage de la programmation utilisant Python ou Javascript.
  Pourquoi Python?  Parce que sa syntaxe est la plus simple et qu'il force des bonnes habitudes d'organisation
  de programmes par le biais de l'indentation obligatoire des blocs de code.
  Pourquoi Javascript?  La raison principale est que c'est le langage utilisé par tous
  les navigateurs du web.  Puisque le logiciel de Reeborg est lui-même écrit en Javascript,
  vous pourrez, après un certain temps, changer la façon dont Reeborg se comporte
  et pourrez même possiblement permettre à Reeborg de faire des choses que je n'aurais
  pas cru possible moi-même!


 J'espère que vous vous amuserez en apprenant la programmation avec Reeborg.
